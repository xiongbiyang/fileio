# ToolPort 部署清单与高阶架构避坑指南 (Deployment & Security Checklist)

> **文档定位**：本指南主要面向主导实际代码开发的研发工程师、全栈开发者或独立创造者（Indie Maker）。其核心作用是封堵在产品代码实现过程中，极易被忽略的非业务层“致命漏洞”与“隐形泥潭”（如资损窃取、安全劫持、工期崩溃），保障 MVP 平稳落地起飞。

---

## 避坑 1：致命业务冲突 —— 劫持防护 (Clickjacking) vs SEO 挂载

我们在产品的增长与安全设定上，埋下了一个不可调和的**底层互斥**：
*   **安全死穴 (工具 C/A)**：作为一个强隐私、免登录的高端传文件与剪贴板工具，如果前端不加防备，黑客能用 CSS `opacity: 0` 的属性将其嵌在一个“领红包”的钓鱼网页顶层。当用户输入微信密码点击领取时，其实内容已经敲进了我们被劫持的共享剪贴板，随后被黑客监视提走（此即臭名昭著的 Clickjacking 点击劫持漏洞），这会瞬间毁掉平台的声誉底线。
*   **流量命脉 (工具 B)**：但为了 SEO 流量，我们又极为渴望小商家将工具 B（二维码）以 `<iframe src=".../embed/qr">` 的形式植入他们的网页，以此薅取高权重外链。

**【强制解决方案】：边缘路由级安全切片 (Cloudflare _headers)**
严禁在前端代码或 Nginx 里做一刀切的全局响应头配置。针对使用 Cloudflare Pages 托管，请在项目根目录强制建立 `_headers` 配置文件，精确打下如下分轨防毒指令：

```text
# 1. 【全局默认锁死】：全域范围内铁壁防御，坚决斩断任何第三方域名的跨站嵌入图谋
/*
  X-Frame-Options: DENY
  Content-Security-Policy: frame-ancestors 'none';

# 2. 【引流专属豁免区】：仅允许我们准备好的 SEO Embed 阉割版微件被任意网站嵌用
/embed/qr/*
  Content-Security-Policy: frame-ancestors *;
  # 切记：此特权路由下不可挂载 X-Frame-Options，因为该旧协定不支持通配，会被上方 CSP 覆盖打通
```

## 避坑 2：大额资损崩盘 —— TURN 打洞凭证“裸奔”

为确保跨网络（非局域网）大体积文件传输不掉线，我们在架构里使用了 Cloudflare TURN 中继（Cloudflare Calls）给 WebRTC 穿透提供保底支持。

*   **常规致死陷阱**：前端开发图省事，将 Cloudflare 签发的原生访问 Token / Secret 直接明文或假装加密地写在了 Vue / Nuxt 的调用层文件里传给浏览器。
*   **灾难后果**：现代前端是开源透明的。黑产脚本只要 F12 扒出您的这个长效密钥，就能把您的 Cloudflare 实例当成他们全网薅羊毛的高速匿名代理（Proxy）隧道，一晚上就可以刷爆绑定在平台背后的信用卡。

**【强制解决方案】：时效性一次性票据 (Ephemeral Credentials)**
1.  **前端严禁触碰主 Key**：前端代码内任何位置都不准出现 Cloudflare API Token。
2.  **后端中转**：发挥 Nuxt 3 的 Server 侧（Nitro）或 PartyKit 的后端环境优势，建一个私默的内部拉取接口 `GET /api/get-turn-auth`。
3.  **秒级抛弃机制**：访客在需要传输前调用此 API，后端用绝对保密的物理系统私钥唤醒 Cloudflare 代理节点，要回一组**生命周期被死死卡在仅“3~5分钟”的临时握手凭证 (Ephemeral Credentials)** 然后返回给客户端供 WebRTC 使用。即使这组凭证被黑客抓包截获，拿到的也是一串转瞬即逝的废纸代码，彻底消灭资损黑洞。

## 避坑 3：情绪与工期耗尽 —— 多语种 (i18n) 大跃进

蓝图中规划的 8 国语言本地化 + pSEO 长尾搜索阵列是一个非常恢弘的流量池规划。对于一个人干全栈的极简创业团队而言，过早铺大摊子等于自我毁灭：

*   **隐形耗损**：每在 UI 上新加一颗“取消”按钮，或修改一条“重新扫码”的失败拦截词，意味着要去维护 8 个深不见底的对应 JSON 翻译表。这在高频试错迭代的 MVP 第一周简直是情绪绞肉机。

**【强制解决方案】：收口主线与 AI 流水线介入**
*   **极简首发 (MVP Week 1)**：严格锁死野心，强行将首发版本缩在 **英语 (English，吃全球流量基本盘)** + **简体中文 (zh-CN，吃国内验证流量盘)** 两个语种。快速打通交互回路并拿到第一波真正的用户使用行为监控数据。
*   **自动化外扩 (V1.2 以后)**：等业务和 UI 样式趋于平稳（且成功提现第一笔 AdSense 变现美元）后，彻底抛弃人工操作。写一个基础的 Node.js 结合 OpenAI / DeepSeek API 的流水线脚本，每次推送前，以 `en.json` 的主键增量比对剩下那几国的 JSON 文件，让 AI 自主在后台完成空缺文案翻译补偿再打包。
