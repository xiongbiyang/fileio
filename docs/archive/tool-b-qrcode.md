## B2. 核心特性

| 特性               | 说明                                                                 |
| :----------------- | :------------------------------------------------------------------- |
| **纯本地处理**     | 所有生成和解析均在浏览器完成，零网络请求，隐私零风险。               |

| **双向功能**       | 同一页面同时提供"文本 → 二维码"和"二维码 → 文本"两个功能区。       |
| **即时响应**       | 输入即生成，无需点击按钮（debounce 300ms），所见即所得。             |
| **多格式下载**     | 生成的二维码支持 PNG / SVG 两种格式下载。                            |

## B3. 技术选型

| 模块           | 方案                                                                     |
| :------------- | :----------------------------------------------------------------------- |
| **二维码生成** | `qrcode`（npm 包），支持 Canvas 和 SVG 双输出，轻量 ~30KB gzip。        |
| **二维码解析** | `jsQR` 或 `@aspect-build/aspect-qr` 或 `zxing-js/library`（上线前评估维护状态与包体积，优先选择活跃维护的方案）。 |
| **图片处理**   | Canvas API 读取上传图片像素数据，传入 jsQR 解码。                        |
| **摄像头扫码** | `navigator.mediaDevices.getUserMedia` + 逐帧 Canvas 解码（移动端增强）。 |
| **文件读取**   | File API + FileReader，支持拖拽、点击选择、粘贴三种方式。                |
| **下载导出**   | Canvas `toBlob()` 导出 PNG；SVG 直接序列化为 Blob 下载。                |

## B4. 功能模块

### B4.1 生成模式（文本 → 二维码）

**输入区域：**
* 多行文本输入框（`<textarea>`），容量上限**随纠错等级动态变化**（见下表）。
* 实时字节计数 + 容量百分比进度条，接近上限时变为警告色。
* 输入变化后 debounce 300ms 自动重新生成，无需手动点击。**注意：** 对于拼音等中文输入法 (IME)，必须利用 `compositionstart` 和 `compositionend` 事件挂起更新，等待候选词落盘后再生成，防止打字过程闪屏。

**QR Code 容量上限（Byte 模式，中文/Emoji/UTF-8 均走此模式）：**

| 纠错等级 | 最大字节数 | 约等于中文字符数 |
| :------- | :--------- | :--------------- |
| L (7%)   | 2,953      | ~984 字          |
| M (15%)  | 2,331      | ~777 字          |
| Q (25%)  | 1,663      | ~554 字          |
| H (30%)  | 1,273      | ~424 字          |

> 注：纯数字模式最大 7,089 位，纯字母数字模式最大 4,296 位，但实际用户输入通常包含中文或特殊字符，统一按 Byte 模式计算。

**生成选项：**

| 选项           | 默认值       | 可选值                           | 说明                                     |
| :------------- | :----------- | :------------------------------- | :--------------------------------------- |
| **尺寸**       | 256×256 px   | 128 / 256 / 512 / 1024 px       | 下载尺寸；页面预览固定 256px。           |
| **纠错等级**   | M (15%)      | L (7%) / M (15%) / Q (25%) / H (30%) | 等级越高容错越强但容量越小。        |
| **前景色**     | `#000000`    | 任意颜色                         | 颜色选择器。**体验防呆**：前端需计算前景与背景对比度，若低于 `3:1` 即导致扫码失效，须在面板闪现警告"⚠️ 颜色对比度过低，将无法被识别"。 |
| **背景色**     | `#FFFFFF`    | 任意颜色                         | 颜色选择器。                             |
| **中心 Logo**  | 无           | 上传图片                         | 叠加在二维码中心，需自动提升纠错等级至 H。（V2.1） |

**输出区域：**
* 实时渲染二维码预览（Canvas）。
* 下方两个按钮：**下载 PNG** / **下载 SVG**。
* **复制到剪贴板** 按钮（`ClipboardItem` + `image/png`）。运行时检测 `typeof ClipboardItem !== 'undefined'`，不支持时（如 Firefox 默认配置）隐藏此按钮，仅保留下载按钮。

**快捷预设（可选折叠面板）：**

| 预设       | 自动填充格式                                                   | 阶段   |
| :--------- | :------------------------------------------------------------- | :----- |
| **纯文本** | 直接输入，默认模式。                                           | MVP    |
| **URL**    | 检测 `http(s)://` 前缀，无前缀自动补 `https://`。             | MVP    |
| **Wi-Fi**  | 表单输入 SSID + 密码 + 加密方式 → `WIFI:T:WPA...` | MVP    |
| **名片(vCard)** | 表单输入姓名与联系方式 → `BEGIN:VCARD...`     | MVP    |
| **Email**  | 表单输入收件人 + 主题 + 正文 → `mailto:...`   | V2.1   |
| **电话**   | 表单输入号码 → `tel:+86...`                    | V2.1   |

> 快捷预设本质是帮用户跑通特定格式的字符串组装。强烈建议**将 Wi-Fi 和名片(vCard) 放入 MVP 阶段起步**。这只需几十行代码但能截获极为庞大的天然搜索长尾流量（如 `wifi qr generator`）。

### B4.2 读取模式（二维码 → 文本）

**输入方式（三选一）：**

| 方式             | 交互                                                                   |
| :--------------- | :--------------------------------------------------------------------- |
| **拖拽上传**     | `FileDropZone` 组件，支持拖入或点击选择文件。                          |
| **粘贴图片**     | 监听 `paste` 事件，从剪贴板读取图片。                                  |
| **摄像头扫码**   | 移动端点击"拍照识别"按钮，打开后置摄像头实时扫描；PC 端显示为"打开摄像头"。按钮文案避免与手机系统"扫一扫"混淆，强调的是"从镜头画面中识别二维码"而非"像微信一样扫码"。 |

**支持格式：** PNG / JPG / JPEG / WebP / BMP / GIF（取第一帧）。

**解析流程：**
1. 用户提供图片 → `FileReader.readAsDataURL()` → 加载到 `<img>`。
2. 绘制到离屏 `<canvas>` → `ctx.getImageData()` 提取像素数据。
3. 传入 `jsQR(imageData.data, width, height)` 解码。
4. 成功：展示解码文本 + **一键复制**。当内容为纯文本时，按钮旁直接悬浮转化入口：“想直接将这段文字在所有设备间留存？👉 [发送到共享记事本(工具C)]”。
5. 失败/多图干扰：当识别失败时，不应直接报错，而是浮现一个“裁切框”UI，提示“图片较复杂，请框选二维码区域”，使用户能手动剔除干扰背景，提高 jsQR 的成功率。

**摄像头扫码流程：**
1. 请求摄像头权限 `getUserMedia({ video: { facingMode: 'environment', advanced: [{ focusMode: 'continuous' }] } })`（增强移动端自动对焦容错）。
2. 视频流渲染到 `<video>` → 每 200ms 截帧到 Canvas → `jsQR` 解码。
3. 识别成功：震动反馈 `navigator.vibrate?.(200)`（iOS 不支持，静默跳过）+ 停止摄像头 + 展示结果。
4. 超时 30s 未识别：提示切换到上传模式。

**解码结果智能识别：**

| 内容类型 | 检测规则               | 额外操作                           |
| :------- | :--------------------- | :--------------------------------- |
| **URL**  | 匹配 `https?://`       | 展示可点击链接 + 复制按钮。        |
| **Wi-Fi**| 匹配 `WIFI:T:...`     | 解析并格式化展示 SSID / 密码。     |
| **Email**| 匹配 `mailto:`        | 展示邮箱地址 + 打开邮件客户端。    |
| **电话** | 匹配 `tel:`           | 展示号码 + 拨号链接（移动端）。    |
| **纯文本**| 以上均不匹配          | 直接展示 + 复制按钮。              |

### B5.2 智能交互行为

| 场景                                 | 自动行为                                                                 |

### B5.4 交互反馈

| 操作               | 反馈                                                       |
| :----------------- | :--------------------------------------------------------- |
| 初始空白态         | 二维码预览区显示灰色虚线框 + 文字"输入文本后自动生成二维码"，而非空白。 |
| 输入文本           | 300ms 后二维码平滑更新（淡入动画，从占位态渐变为二维码）。 |
| 超出容量限制       | 输入框边框变红 + 计数器变红 + 二维码区域显示警告。         |
| 高密度矩阵隐患     | 当文本极长（如>300字）但未超限时，骤增的矩阵块使扫码变难。此时二维码下方必须动态闪现黄字警告："⚠️ 内容过多导致矩阵极其密集，中低端摄像头可能难以识别，建议缩短"。 |
| 下载成功           | 按钮短暂变绿 + "已下载"提示。                              |
| 复制成功           | 按钮短暂变绿 + "已复制"提示。                              |
| 摄像头扫码识别成功 | 震动 + 扫描框变绿 + 自动切到结果页。                       |
| 解码失败           | 图片预览区叠加半透明红色蒙版 + 错误提示。                  |

## B9. 安全与防滥用

| 机制               | 规则                                                                     |
| :----------------- | :----------------------------------------------------------------------- |
| **纯本地处理**     | 全部计算在浏览器完成，不存在服务端数据泄露风险。                         |
| **图片大小限制**   | 上传图片最大 10MB，防止前端内存溢出。                                    |
| **Canvas 安全**    | 解析仅处理用户主动提供的图片，不存在跨域 Canvas 污染风险。               |
| **摄像头权限**     | 仅在用户主动点击"扫一扫"时请求，关闭后立即释放 MediaStream。            |
| **恶意内容防护**   | 解码结果中的 URL 不自动跳转，仅展示为可点击链接，用户自行判断是否访问。  |
